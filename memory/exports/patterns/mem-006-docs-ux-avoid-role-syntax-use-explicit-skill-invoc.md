# Docs UX: Avoid @Role syntax; use explicit skill invocation

## Summary
Developer-facing docs should not require @PM/@Reviewer mentions (which can conflict with editors). Prefer explicit skill names (e.g. 'reviewer: run Stage 3 review') and describe optional tool integrations without backward-compat role syntax.
